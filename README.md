# Автоматизация логистики и учёта белья между отелями и прачечной

## Стек и инструменты

`Business Analysis` `System Analysis` `AS IS / TO BE` `Use Case` `BPMN` `UML` `Functional Requirements` `Non-functional Requirements` `REST API` `JSON` `Acceptance Testing` `Project Documentation`

---

## О проекте

Проект посвящён доработке системы **«Л-логистика»** для сети отелей **Oasis Retreats**.

Система используется для планирования доставки и забора белья между отелями и прачечной. В рамках проекта нужно было разобраться в текущем процессе, выявить проблемы ручного учёта и предложить решение, которое сделает логистику белья более прозрачной, управляемой и предсказуемой.

Моя роль в проекте — **бизнес-аналитик**.

---

## Контекст задачи

В текущем процессе отели и прачечная работали с большим количеством ручных операций:

- остатки белья фиксировались в Excel и ручных записях;
- потребность в белье рассчитывалась сотрудниками вручную;
- данные о загрузке отеля из PMS не использовались автоматически;
- отели и прачечная вели разрозненные таблицы;
- фактические объёмы поставок, заборов и расхождения было сложно контролировать;
- не было единого источника актуальных данных по остаткам, поставкам и истории операций.

Из-за этого возникали ошибки в учёте, риск дефицита белья, лишние коммуникации между сотрудниками и низкая прозрачность процесса.

---

## Цель проекта

Разработать аналитическое решение для доработки системы **«Л-логистика»**, которое позволит:

- автоматизировать расчёт потребности в белье;
- использовать данные PMS **«О! Отель»** при планировании поставок;
- снизить количество ручных операций и ошибок;
- сделать движение белья между отелем и прачечной прозрачным;
- фиксировать историю поставок, заборов, инвентаризаций и списаний;
- подготовить требования и документацию для дальнейшей разработки.

## Предложенное решение

В целевом решении система **«Л-логистика»** должна стать единым рабочим инструментом для логиста и сотрудников отеля.

Ключевые возможности решения:

- автоматический расчёт потребности в белье на основе загрузки отеля;
- получение данных из PMS **«О! Отель»**;
- формирование плана поставок по каждому отелю;
- ручная корректировка рассчитанного плана;
- утверждение плана поставок;
- контроль статусов поставки;
- учёт остатков белья;
- фиксация инвентаризаций и списаний;
- просмотр истории поставок и заборов;
- формирование отчётов по поставкам за период;
- журналирование действий пользователей.
